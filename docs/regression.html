Ivory: Experimental Results

This page describes a number of experiments on standard test collections using Ivory. Each experiment is associated with a command-line invocation to perform the experiment, command-line invocations to evaluate the results, and a JUnit test case that performs the experiment and checks to see if the results are as expected (in terms of standard metrics of effectiveness). The purpose of enumerating these experiments is for others to be able to replicate our results, thereby providing a solid foundation for building on our work.

Here are the experiments categorized:

  • Experiments on TREC Disks 4-5: baselines, weighted sequential dependence model, latent concept expansion
  • Experiments on Wt10g: baselines
  • Experiments on Gov2: baselines
  • Experiments on first segment of ClueWeb09 (i.e., TREC Web09 track, category B): baselines, incorporating Waterloo spam scores
  • Experiments from Wang et al.'s SIGIR 2010 paper: the efficient sequential dependence model (ESD) on Wt10g, Gov2, and Clue

Basic + LCE models

LCE refers to Metzler et al's Latent Concept Expansion model (SIGIR 2007). These models take a long time to run (approximately 1.5 hours on our production machine).

# command-line
etc/run.sh ivory.smrf.retrieval.RunQueryLocal docs/data/trec/run.robust04.basic.lce.xml docs/data/trec/queries.robust04.xml

# evaluating effectiveness
trec_eval docs/data/trec/qrels.robust04.noCRFR.txt ranking.robust04-dir-rm3-f.txt
trec_eval docs/data/trec/qrels.robust04.noCRFR.txt ranking.robust04-dir-rm3-s.txt
trec_eval docs/data/trec/qrels.robust04.noCRFR.txt ranking.robust04-dir-sd-lce-f.txt

Baseline + Waterloo spam scores

The first two runs represent baseline QL with the addition of the Waterloo spam scores as document-level features. The two settings represent the best MAP and best P10 from a parameter sweep (note, no cross-validation was performed). The third run is a parameter sweep with bm25: this was the setting found to have best MAP and P10.

# command-line
etc/run.sh ivory.smrf.retrieval.RunQueryLocal docs/data/clue/run.web09catB.spam.xml \
  docs/data/clue/queries.web09.xml

# evaluating effectiveness
trec_eval docs/data/clue/qrels.web09catB.txt ranking.web09catB-ql-waterloo-spam.0.3.txt 
trec_eval docs/data/clue/qrels.web09catB.txt ranking.web09catB-ql-waterloo-spam.0.5.txt 
trec_eval docs/data/clue/qrels.web09catB.txt ranking.web09catB-bm25-waterloo-spam.0.02.txt 

# junit
etc/junit.sh ivory.regression.Web09catB_Baseline_WaterlooSpam
description tag MAP P10
QL + Waterloo spam scores (weight 0.3) web09catB-ql-waterloo-spam-0.3 0.2163 0.4220
QL + Waterloo spam scores (weight 0.5) web09catB-ql-waterloo-spam-0.5 0.2143 0.4540
bm25 + Waterloo spam scores (weight 0.02) web09catB-bm25-waterloo-spam-0.02 0.2180 0.4240

Results from SIGIR 2010

These regression runs represent experiments presented in:

Lidan Wang, Jimmy Lin, and Donald Metzler. Learning to Efficiently Rank. Proceedings of the 33rd Annual International ACM SIGIR Conference on Research and Development in Information Retrieval (SIGIR 2010), July 2010, Geneva, Switzerland.

Note that Bendersky et al.'s WSD model (WSDM 2010) is a special case of the ESD model (without any time constraints). These runs represent results in Tables 5 and 6 of the paper.

Wt10g results

# command-line
etc/run.sh ivory.smrf.retrieval.RunQueryLocal docs/data/wt10g/run.wt10g.SIGIR2010.xml \
  docs/data/wt10g/wt10g_queries_501-550.xml

# evaluating effectiveness
trec_eval docs/data/wt10g/qrels.wt10g ranking.sigir2010-wt10g-ql.txt
trec_eval docs/data/wt10g/qrels.wt10g ranking.sigir2010-wt10g-sd.txt
trec_eval docs/data/wt10g/qrels.wt10g ranking.sigir2010-wt10g-wsd-sd.txt

# junit
etc/junit.sh ivory.regression.Wt10g_SIGIR2010
description tag MAP P10
Dirichlet, full independence wt10g-ql 0.2151 0.3560
Dirichlet, sequential dependence wt10g-dir-sd 0.2242 0.3640
Dirichlet, ESD, no time constraints (=WSD) wt10g-wsd-sd 0.2411 0.3820
